    1. 创建TypeService，TypeServiceImpl，TypeRepostory

      @Service
@Transactional
public class TypeServiceImpl implements TypeService {

    @Autowired
    private TypeRepository typeRepository;


      @Override
      public Type saveType(Type type) {
          return typeRepository.save(type);
      }

      @Override
      public Type getType(Long id) {
          return typeRepository.getOne(id);
      }

      @Override
      /*
      pageable是springboot自带的分页工具
      */
      public Page<Type> listType(Pageable pageable) {
          return typeRepository.findAll(pageable);
      }

      @Override
      public Type updateType(Long id, Type type) {
          Type t = typeRepository.findOne(id);//查询到分类
          if(t == null){
              throw new NotFindException("不存在该分类！");
          }
          BeanUtils.copyProperties(type,t);//将分类覆盖到查到的对象中，实现更新
          return typeRepository.save(t);//将其保存到数据库
      }

      @Override
      public void deleteType(Long id) {
          typeRepository.delete(id);
      }
  }
  ```

  3. 增加校验 分类不能重复

    1. 增加service/dao方法，通过名字查询分类getTypeByName(String name)

    2. 在controller通过type.getName的名字查询，增加错误消息

          @PostMapping("/types")
    public String post(@Valid Type type,//表单提交的name会封装进type对象中
                       BindingResult result, //绑定校验结果，用于保存错误信息
                       RedirectAttributes attributes){
        //校验分类不能重复
        Type type1 = typeService.getTypeByName(type.getName());
        if(type1 != null){
            result.rejectValue("name","nameError","该分类已存在");
            //参数1：属性名，参数2：自定义，参数3：提示消息
            //作用：增加错误信息
        }
        //校验name不能为空
        if(result.hasErrors()){
            return "/admin/types-input";
        }
        //保存操作
        Type t = typeService.saveType(type);
        if(t == null){//保存失败
            attributes.addFlashAttribute("message","操作失败");
        }else{//保存成功
            attributes.addFlashAttribute("message","操作成功");
        }
        return "redirect:/admin/types";
    }

      遇到的坑：

      分类已经存在，但仍然保存成功

      原因：先判断了是否有错误信息（检验不为空），再添加了错误信息（校验不重复）

      解决：先校验不重复，添加错误信息，再校验不为空，判断是否有错误信息，有就跳转

    3. 消息会显示在上个验证name的提示框中，所以直接测试即可

    2. 前端页面types-input.html

      1. 三元判断action，判断是否有id，来确定是保存还是修改页面
      <form action="#" method="post" th:object="${type}" th:action="*{id}==null?@{/admin/types}:@{/admin/types/{id}(id=*{id})}" class="ui form"><!--这里的id是update方法中第一个参数-->

    1. 无法完成跳转，也没有更新成功

    1. controller：delete(Long id)

         @GetMapping("/types/{id}/delete")
    public String delete(@PathVariable Long id,RedirectAttributes attributes){
        typeService.deleteType(id);
        attributes.addFlashAttribute("message","删除成功");

      遇到的坑：

      404/500错误

      原因：不能使用转发的方式，应该用重定向

    2. 优化：分类中有博客会导致删除失败

      1. service增加按照标签id查找博客方法

        public List<Blog> listBlog(Long typeId) {
    return blogRepository.findAll(new Specification<Blog>() {
        @Override
        public Predicate toPredicate(Root<Blog> root, CriteriaQuery<?> cq, CriteriaBuilder cb) {
            Join join = root.join("type");
            return cb.equal(join.get("id"),typeId);
        }
    });
}
      @GetMapping("/types/{id}/delete")
public String delete(@PathVariable Long id,//url传过来的id
                     RedirectAttributes attributes){//保存提示消息
    List<Blog> blogs = blogService.listBlog(id);//查询type下的所有博客
    if(blogs.size()!=0){
        //证明分类中有博客
        attributes.addFlashAttribute("message","删除失败，请先删除分类中的博客！");
        return "redirect:/admin/types";
    }
    typeService.deleteType(id);
    attributes.addFlashAttribute("message","删除成功");
    return "redirect:/admin/types";
}
