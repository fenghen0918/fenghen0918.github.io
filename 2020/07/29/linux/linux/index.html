Linux常见命令

总结常见命令

查看环境变量值：echo $PATH（查看path变量）

检查gcc的版本：gcc -v

检查文件夹大小：du -sh 文件名

清空文件内容（用于清空日志）：cp /dev/null catalina.out

重新加载环境变量：source /etc/profile

查看操作系统位数：uname -a，如果信息中有x86_64，则是64位，没有就是32位

动态查看日志：tail -f catalina.out，如果查看后100行：-100f

列出文件列表：ls（查看文件名列表） ll（查看文件列表详细信息）

ls -a显示所有文件、隐藏文件

ll -h显示所有文件信息，文件大小可读性较高

ll -t按创建时间倒序（正序-tr）排列

du -sh 目录路径显示目录的真实大小

路径

./ ：当前目录

../：上一级目录

显示当前目录路径 pwd

切换目录命令cd：

  • cd ..（返回上一级）
  • cd /（返回根目录）
  • cd ~（返回用户主目录）
  • cd - （返回上一次所在目录）
  • cd /usr/文档（进入目标目录）

创建目录和移除目录：

  • mkdir（创建文件夹） mkdir -p aaa/bbb（创建多级目录）
  • rmdir（移除文件夹）只能删除空文件夹

浏览文件：

  • cat（了解）、more、less（多个上下键功能）：回车显示下一行内容，空格显示下一页内容，按 q 键退出查看

  • tail：用于显示文件后几行的内容 ctrl+c结束查看

    • 用法:

      tail -10 /etc/passwd  查看后10行数据

      tail -f catalina.log  动态查看日志(*****)

  • head -10 文件路径：查看文件的前10行

  • wc 文件路径：统计文件的行数、字数、字节数

文件操作：../上一级目录 ./当前目录

复制：cp 剪切：mv

  • 用法：

    cp xx.txt aa 复制到aa文件夹中

    cp xx.txt aa/yy.txt 复制到aa文件夹中，重命名为yy.txt

    选项：

    -r： cp -r aa /aa 复制文件夹aa到根目录下，-r递归复制，加上可以复制文件夹

    -i： cp -i aa /aa 复制文件夹到根目录下，若目标档已存在，先询问动作再执行

    -p： 连同文件的属性也复制过去，而非使用默认属性（备份常用）

    -f： 强制复制，如果目标文件已存在且无法开启，移除再尝试一次

删除：rm

  • 用法：

    rm -r aa 可删除文件夹

    rm -rf aa 不询问删除

打包、解压：tar（打包文件：tar，被压缩的打包文件：tar.gz）

  • 常用参数：

    -c：创建一个新tar文件

    -v：显示运行过程的信息

    -f：指定文件名

    -z：调用gzip压缩命令进行压缩

    -t：查看压缩文件的内容

    -x：解开tar文件

    打包：

    ​ tar –cvf xxx.tar ./* 将当前目录下的文件打包

    打包并且压缩：

    ​ tar –zcvf xxx.tar.gz ./*

    解压

    ​ tar –xvf xxx.tar

    ​ tar -zxvf xxx.tar.gz -C /usr/aaa（将解压的结果放在aaa目录下）

查找：

  • find：查找符合条件的文件

    • 示例：

      find / -name ins* 查找文件名称是以ins开头的文件

      find / -name ins* –ls

      find / –user itcast –ls 查找用户itcast的文件

      find / -type f 查找根目录下的所有文件（f代表文件，d代表文件夹）

  • grep：查找文件里符合条件的字符串

    • 示例：

      grep lang anaconda-ks.cfg 在文件中查找lang

      grep lang anaconda-ks.cfg –color 高亮显示

      grep lang anaconda-ks.cfg –color -A5 相关字符串前五行显示

查看时间 date

date：输出形式 2021年3月21日 星期日 23:42:28

date +%F：输出形式 2021-03-21

date “+%F %T”：输出形式 2021-03-21 23:42:28

date -s “20220810 22:25:35”

cal：查看当前日历

内存

  1. free -mh：查看内存使用情况
  2. sync：将缓存写入文件系统
  3. echo 3 > /proc/sys/vm/drop_caches：释放内存
  4. 再次查看内存情况
  5. echo 0 > /proc/sys/vm/drop_caches：恢复设置

　　0：0是系统默认值，默认情况下表示不释放内存，由操作系统自动管理

　　1：释放页缓存

　　2：释放dentries和inodes

　　3：释放所有缓存

系统管理命令 ps（进程）

  • ps –ef 查看所有进程
    • uid：用户id
    • pid：进程id
    • ppid：父级进程id
    • c：cpu占用率
    • stime：进程启动时间
    • tty：终端设备，发起该进程的设备识别符号，？表示系统自动的
    • time：进程执行时间
    • cmd：该进程名称/路径
  • ps –ef | grep ssh 查找含有ssh的进程
    • ef：系统所有进程信息
    • ux：当前用户进程信息
    • aux：所有用户进程信息
  • top（任务管理器）
    • pid：进程id
    • user：用户
    • pr：优先级
    • virt：虚拟内存
    • res：常驻内存
    • shr：共享内存（实际内存=常驻内存-共享内存）
    • s：进程状态（s表示睡眠，r表示运行）
    • %cpu：表示cpu占用比
    • %mem：内存占用比
    • time：执行时间
    • command：进程名称/路径
  • top的快捷键
    • M：按内存从高到低排序
    • P：CPU从高到低
    • 1：查看每个cpu的使用情况
  • kill 2868 杀掉2868编号的进程
  • kill all 进程名称
  • kill -9 2868 强制杀死进程
  • pkill -9 进程名 ：这样杀掉更干净

防火墙根据配置文件/etc/sysconfig/iptables来控制本机的”出”、”入”网络访问行为。
service iptables status 查看防火墙状态
service iptables stop 关闭防火墙 ***
service iptables start 启动防火墙
chkconfig  iptables off 禁止防火墙自启

centos7
关闭防火墙的方法：systemctl stop firewalld
启动：systemctl start firewalld
查看状态：systemctl status firewalld
开机禁用：systemctl disable firewalld
开机启用：systemctl enable firewalld
*增加防火墙端口（开放一个端口可访问）：firewalld-cmd --zone=public --add-port=80/tcp --permanent (--permanent 永久生效，没有该参数重启后失效)
重新载入（刷新配置）：firewall-cmd --reload
查看所有打开的端口：firewall-cmd --zone=public --list-ports
删除端口：firewall-cmd --zone= public --remove-port=80/tcp --permanent

export PATH=$PATH:${JAVA_PATH}:$PATH

export CATALINA_HOME=/opt/tomcat7
export CATALINA_BASE=/opt/tomcat7
export TOMCAT_HOME=/opt/tomcat7
export CATALINA_HOME2=/opt/tomcat8
export CATALINA_BASE2=/opt/tomcat8
export TOMCAT_HOME2=/opt/tomcat8

    让环境变量生效，source /etc/profile

    redis是C语言开发，安装redis需要先将官网下载的源码进行编译，编译依赖gcc环境。

    输入命令：

    yum install gcc-c++

  2. 下载redis源码

    wget http://download.redis.io/releases/redis-3.0.4.tar.gz

  3. 解压

    tar -xzvf redis-3.0.4.tar.gz

  4. 编译

    切换都程序目录，并执行make命令编译

    cd redis-3.0.4

    make

  5. 安装

    将编译好的文件安装成可执行文件

    make PREFIX=/usr/local/redis install 

    make install安装完成后，会在/usr/local/bin目录下生成下面几个可执行文件，它们的作用分别是：
redis-server：Redis服务器端启动程序
redis-cli：Redis客户端操作工具。也可以用telnet根据其纯文本协议来操作
redis-benchmark：Redis性能测试工具
redis-check-aof：数据修复工具
redis-check-dump：检查导出工具

  6. 配置redis

    复制配置文件到/usr/local/redis/bin目录：

    cd redis-3.0.4

    cp redis.conf /usr/local/redis/bin

  7. 启动redis

    1. 进入redis/bin目录

      cd redis/bin

    2. 启动redis服务端

      ./redis-server redis.conf

    3. 克隆新窗口，启动redis客户端

      ./redis-cli

  8. 操作redis

    keys *
set name zhangsan
get zhangsan

远程连接mysql

linux的mysql只是当作一个服务器来使用，windows想要连接linux上的Mysql，我们常用的图形化界面sqlyog

注意：连接时会报错，报错代码2003，说明我们linux防火墙没有关闭。

  1. 关闭防火墙
    • centos7关闭防火墙的方法：systemctl stop firewalld
    • centos6及以下关闭防火墙的方法：service iptables stop

注意：如果报错1045，说明我们需要授权本机电脑为root权限

  1. 授权

    grant all privileges on *.* to 'root' @'%' identified by '你的密码';
flush privileges; 刷新，立即生效

# 这里的root代表root用户，最后的root代表root用户的密码，我这里用户名，密码都是root，中间的%号代表所有的ip地址都能远程访问，如果你要针对某一台机器开放远程访问权限，那就把%换成哪台机器的ip地址，这样就只能这台机器远程访问linux下的mysql数据库了。

  2. 连接成功

Linux自有服务

运行模式

指系统的运行模式，也称运行级别

进程：init 进程id为1。对应的进程配置文件：inittab（/etc/inittab）

0-关机模式
1-单用户模式
2-多用户模式，不带NFS
3-多用户模式，完全
4-没有被使用的模式
5-x11，完整图形化界面模式
6-重启
  1. init 0 表示关机
  2. init 3 切换不带桌面模式
  3. init 5 切换图形界面
  4. Init 6 重启电脑

永久切换命令行模式：

​ 将/etc/inittab文件的initdefault设置成3，然后重启系统。

0-6表示上面的运行模式中是否开启或关闭

用户管理

添加用户

常用语法：useradd 选项 用户名

常用选项：

  • -g：表示指定用户的用户组，选项的值可以是Id，也可以是组名
  • -G：表示指定用户的附加组，选项的值可以是Id，也可以是组名
  • -u：uid，用户的id
  • useradd -g 501 -G 500 -u 666 lisi李四的用户组501，附加组500，uid为666

验证是否成功：

  1. 在/etc/passwd最后一行，查看zhangsan的信息
  2. 验证是否存在家目录：ls /home/

修改用户

常用语法：usermod 选项 用户名

常用选项：

  • 上面3个，和-l：修改用户名

usermod -l wangwu zhangsan：把张三改为王五

su - root的-代表该表用户变量

常用选项:

  • -r ：表示删除用户的同时，删除其家目录

  • -n：修改用户组组名

用户组删除

sudo，非root用户执行root操作

Sudo配置文件：/etc/sudoers，配置请使用visudo

all 表示允许登录的主机
(all) 表示以谁的身份执行 all表示root身份
all 表示当前用户可以执行的命令

调整存储分区

df -h查看是否/目录最大，如果不是，通过以下操作使根目录最大。

  1. 备份home分区文件
    tar cvf /tmp/home.tar /home

  2. 卸载/home，如果无法卸载，先终止使用/home文件系统的进程
    umount /home （卸载）

    卸载时，发现/home在使用中，所以先终止。
    fuser -km /home/（终止）
    再次卸载，没有报错，表示成功。

  3. 删除/home所在的lv

    lvremove /dev/mapper/cl-home

  4. 扩展/root所在的lv
    lvextend -L +460G /dev/mapper/cl-root （多少G，具体看自己）

  5. 扩展/root文件系统
    xfs_growfs /dev/mapper/cl-root

  6. 创建/home所在的lv
    lvcreate -L 50G -n home cl

  7. 创建文件系统
    mkfs.xfs /dev/mapper/cl-home

  8. 挂载home
    mount /dev/mapper/cl-home

  9. home文件恢复
    tar xvf /tmp/home.tar -C /home/
