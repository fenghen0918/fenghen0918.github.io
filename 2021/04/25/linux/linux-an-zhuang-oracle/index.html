Linux安装Oracle

前期准备

  1. oracleLinux服务器7.6
  2. oracle11.2.0.4数据库安装包

网络环境

虚拟机和主机可互相Ping通，能访问外网。

安装环境准备

  1. 修改服务器主机名：hostnamectl set-hostname db (db为主机名)

  2. 修改hosts文件：vim /etc/hosts

    将以下内容补充到配置文件的末尾： < Server IP > < HOSTNAME > 如：192.168.200.124 db

  3. 修改用户环境变量

    1. root用户变量，在文件末尾添加vim /etc/profile

      if [ $USER = "oracle" ]; then
if [ $SHELL = "/bin/ksh" ]; then
ulimit -p 16384
ulimit -n 65536
else
ulimit -u 16384 -n 65536
fi
fi

    2. oracle用户变量

      [oracle@oracle ~]$ su - oracle

      • 没有oracle用户，需要创建该用户

        useradd -u 54321 oracle，组在之后设置

        passwd oracle

        输入新密码

        • -bash: ulimit: max user processes: cannot modify limit: Operation not permitted

          vim /etc/security/limits.conf

          末尾增加如下内容

          oracle soft nproc 2047
oracle hard nproc 16384
oracle soft nofile 1024
oracle hard nofile 65536

      [oracle@oracle ~] cat .bash_profile

        vim .bash_profile
  # .bash_profile
  # Get the aliases and functions
  if [ -f ~/.bashrc ]; then
  . ~/.bashrc
  fi
  # User specific environment and startup programs
  export ORACLE_BASE=/u01/app/oracle
  export ORACLE_HOME=$ORACLE_BASE/product/11.2.0.4/dbhome_1
  export ORACLE_SID=orcl
  export LD_LIBRARY_PATH=$ORACLE_HOME/lib:/lib:/usr/lib
  export
  CLASSPATH=$ORACLE_HOME/JRE:$ORACLE_HOME/jlib:$ORACLE_HOME/rdbms/jlib
  PATH=$PATH:$HOME/.local/bin:$HOME/bin:$ORACLE_HOME/bin
  export PATH

      [oracle@oracle ~]$ source .bash_profile

      安装部署

  4. 配置本地yum源

    方式一：

    由于虚拟机无法连接外网，所以yum在线下载是不可用的，因此我们需要配置本地yum源（若 服务器可以联网，可跳过此不走，直接执行 yum install -y oracle-rdbms-*）

    1. ls -l /dev | grep cdrom，查看是不是cdrom

    2. 如果是，mount /dev/cdrom /media/

    3. 在终端中输入命令vi /etc/yum.repos.d/public-yum-ol6.repo（这里如果是创建了新的文件，就打开目录看看是不是ol6，有可能是ol7，修改ol7即可）， 将 public 配置文件中的“enabled=1”改为“enabled=0”，禁用所有 yum 源引用配置。 并在配置文件末尾输入以下内容：

      [db] 
name=db 
baseurl=file:///media

    4. 导入光盘 KEY 文件至 media 目录，使用命令 rpm --import /media/RPM-GPG-KEY*

    5. yum list测试

    方式二（可用）：

    1. 将oracle linux7.iso镜像文件上传到/opt目录下

    2. 将/etc/yum.repos.d/里的文件删除或移动到别处

    3. 新建文件CentOS7-Base-local.repo，输入

      [db]
name=db
baseurl=file:///mnt/cdrom
enabled=1
gpgcheck=0
gpgkey=file:///mnt/cdrom/RPM-GPG-KEY-oracle

    4. 挂载镜像mount -o loop linux76.iso /media/

    5. 移动挂载目录到mnt/cdrom

      1. cd /media
      2. mkdir /mnt/cdrom
      3. cp -Rv * /mnt/cdrom/

    6. yum list测试

      如果出现Another app is currently holding the yum lock; waiting for it to exit…

      原因：yum进程正在被占用

      解决：关掉yum进程rm -f /var/run/yum.pid

  5. 详细安装

    1. 安装环境准备包

      yum install -y oracle-rdbms-*，提示complete就是完成

      遇到问题：

      如果出现Nothing to do

      解决：yum install -y oracle*

    2. 查看并更改oracle用户密码

      1. id oracle 回车，查看 oracle 用户组；

      2. 修改用户组

        usermod -g oinstall -G dba oracle

        下载完oralce之后，可以查看组，cat /etc/group，发现已经有了oinstall和dba组，所以可以将Oracle用户划分到该组当中

      3. passwd oracle 回车

        上面设置过了就不需要再次设置

    3. 创建 Oracle 11g database 安装目录 u01

      # mkdir /u01

      #chown oracle:oinstall /u01 -R

      对刚建立的 u01 文件夹进行递归修改，赋予读写权限。

    4. 上传安装程序压缩包，上传至/opt目录

    5. 解压安装程序压缩包并修改属主

      # unzip linux.x64_11gR2_database_1of2.zip
# unzip linux.x64_11gR2_database_2of2.zip
# chown oracle:oinstall -R /opt/database/

    6. 启动图形化安装界面

      1. 切换成oracle用户，su - oracle

      2. 输入命令：./runInstaller，如果出现y/n，重启虚拟机，登录界面使用oracle再次尝试

        重新登录oracle时进不去系统时

        先登录root，删除oracle用户，再创建oracle用户，将上面的环境变量和权限重新配置。

        可以使用xshell+xmanager打开可视化界面。避免linux视图显示不全

    7. 安装，具体流程参考pdf

      INS 30131 错误。[INS-30131] Initial setup required for the execution of installer valid

      解决：cd /tmp

      rm -rf CVU_11.2.0.4.0_oracle

      重新执行./runInstaller

      运 行 至 86% 时 会 出 现 报 错 “ Error in invoking target ‘agent nmhs’ of makefile……”

      解决：

      修改$ORACLE_HOME/sysman/lib/ins_emagent.mk，

      将 $(MK_EMAGENT_NMECTL)修改为：$(MK_EMAGENT_NMECTL)-lnnz11，修 改后点击 Retry

      注意：过程中按照步骤执行两个脚本

  6. 配置部分

    1. 创建数据库监听

      [oracle@db ~]# netca

    2. 创建数据库实例

      [oracle@db ~]# dbca

数据库自启配置

  1. 修改/etc/oratab文件

    [root@Oracle11g ~]# vi /etc/oratab找到： orcl=/u01/app/oracle/product/11.1.0/dbhome_1:N 改为： orcl=/u01/app/oracle/product/11.1.0/dbhome_1:Y 也就是把最后的 N 改为 Y，意思是将不允许自动启动改为允许自动启动。

  2. 修改 ORACLE 自带的启动和关闭脚本

    分别是 dbstart 和 dbshut，执行这两个脚本就可以实现 ORACLE 脚本的启动 与关闭。

    在 oracle 用户下修改$ORACLE_HOME/bin/dbstart 文件

    [root@Oracle11g ~]# su – oracle
[oracle@Oracle11g ~]$ cd $ORACLE_HOME/bin
[oracle@Oracle11g bin]$ vi dbstart
找到：
# First argument is used to bring up Oracle Net Listener
ORACLE_HOME_LISTNER=$1
将 ORACLE_HOME_LISTNER=$1
改为：ORACLE_HOME_LISTNER=$ORACLE_HOME

  3. 将脚本注册为系统服务

    1. 创建/lib/systemd/system/oracle11g.service

      [oracle@db bin]$ cat /lib/systemd/system/oracle11g.service

      [Unit]
Description=The Oracle Database Service
After=network.target
[Service]
Type=forking
RemainAfterExit=yes
KillMode=none
TimeoutStopSec=5min
User=oracle
Group=oinstall
Environment="ORACLE_HOME=/u01/app/oracle/product/11.2.0.4/dbhome_1" 
ExecStart=/u01/app/oracle/product/11.2.0.4/dbhome_1/bin/dbstart $ORACLE_HOME &
ExecStop=/u01/app/oracle/product/11.2.0.4/dbhome_1/bin/dbshut $ORACLE_HOME &
Restart=no
[Install]
# Puts wants directive for the other units in the relationship
WantedBy=default.target

      [root@db ~]# systemctl daemon-reload

      [root@db ~]# systemctl start oracle11g.service

      [root@db ~]# systemctl enable oracle11g.service

      完成以上操作后，可以重启系统查看是否是设置成功。

问题：提示错误如下

[root@db ~]# systemctl start oracle11g.service
Job for oracle11g.service failed because the control process exited with error code. See "systemctl status oracle11g.service" and "journalctl -xe" for details.

原因：在保存上面的oracle11g.service配置文件时，:x输入成了:X，设置了密码导致文件加密。

解决：输入密码打开文件，再次输入:X，直接回车两下，将密码设置为空。成功解密，再次尝试

系统容灾策略

数据库备份

本教程可使用 system 账户将整个实例备份（包含空表），并自动删除超过 6 天 的历史数据。

  1. root 用户登录服务器

    mkdir -p /backup/oracledata #新建 Oracle 数据库备份目录
chown -R oracle:oinstall /backup/oracledata #设置目录权限为 oinstall 用
户组的 oracle 用户（用户 oracle 与用户组 oinstall 是在安装 Oracle 数据库时
设置的）

  2. 创建目录对象

    因为使用 expdp 命令需要使用到目录对象(directory)，所以首先需要创建目 录对象，并赋予某个用户对目录对象的操作权限。

    su – oracle #切换到 oracle 用户

    sqlplus / as sysdba #切换到 SQLPLUS 环境

    执行以下命令：

    create directory expdp_bak_dir as '/backup/oracledata'; #指定输出目录

    报错：

    ERROR at line 1:
ORA-01034: ORACLE not available

    原因：一般是由于监听或Oracle没有启动

    1. 首先先测试一下，监听是否启动：lsnrctl status；如果监听没有启动，则 lsnrctl start；

    2. 启动oracle

      1. sqlplus / as sysdba

      2. startup

        报错：LRM-00109: could not open parameter file ‘/u01/app/oracle/product/11.2.0.4/dbhome_1/dbs/initecard.ora’

        原因：看到找不到intecard.ora。记得自己的实例创建时是orcl。有可能是因为变量设置出现问题。实例名ORACLE_SID设置不对

        1. su - oracle
        2. vim .bash_profile
        3. 修改ORACLE_SID=orcl
        4. 重启变量source .bash_profile
        5. 再次尝试启动oracle

  3. 编辑自动备份脚本

    vi /backup/oracledata/oradatabak.sh#新建文件，输入以下代码

    #!/bin/sh
export ORACLE_SID=orcl
export ORACLE_HOME=/u01/app/oracle/product/11.2.0.4/dbhome_1
export NLS_LANG=american_america.al32utf8
export DATA_DIR=/backup/oracledata
now=` date +%Y%m%d%H%M%S ` 
dmpfile=db$now.dmp
echo start exp $dmpfile ... 
/u01/app/oracle/product/11.2.0.4/dbhome_1/bin/expdp system/Jiaminjm1 DUMPFILE=$dmpfile logfile=db$now.log DIRECTORY=expdp_bak_dir full=y
echo "Delete the file bakup before 6 days..."
find /backup/oracledata/ -mtime +6 -name "*.dmp" -exec rm -rf '{}' \;
echo "Delete the file bakup successfully. " 
echo "Bakup completed."

    注意点：

    1. 第二行ORACLE_SID，改为数据库的实例名

    2. 第九行，必须是目录后，空格，用户名/密码

      1. 不能将用户名密码一直到full=y另起一行写，会出错，因为这是一个完整的语句。

        错误信息
/backup/oracledata/oradatabak.sh: line 10: system/Jiaminjm1: No such file or directory

    3. 备份文件目录：/u01/app/oracle/admin/orcl/dpdump/expdat.dmp

    chmod +x /backup/oracledata/oradatabak.sh #添加脚本执行权限

  4. 设置系统自动定时执行备份脚本

    vi /etc/crontab #编辑系统任务计划

    将下面的代码写入最后一行： 00 0 * * * oracle /backup/oracledata/oradatabak.sh

    ./oradatabak.sh 是手动备份

    #每天凌晨 0 点 00 分，以 oracle 用户执行 oradatabak.sh 备份文件

    [root@db ~]# systemctl restart crond.service

    [root@db ~]# systemctl enable crond.service

    即每天凌晨 0 点 00 分进行备份，如需每天备份多次，可设置不同时间段备份：

    例如：* 3,13,18 * * * oracle /backup/oracledata/oradatabak.sh 即每天 3 点、13 点、18 点进行备份。

    说明：文件备份目录，用户oracle必须有更改权限，否则无法备份

    问题：到时间了却没有备份。查看当前时间date，发现系统时间不同步

    解决：

    1. yum -y install ntpdate
    2. ping baidu.com，确定能访问外网
    3. /usr/sbin/ntpdate stdtime.gov.hk同步时间
    4. date查看时间

数据库备份恢复

恢复需在数据库服务器上进行，并保持程序、数据文件路径与备份前一致。

1) 在 PL/SQL 环境，设置数据库导入目录 SQL>create directory impdp_dir as '/backup/oracledata';

2) 在 PL/SQL 环境，授权 SYSTEM 用户访问授权 SQL>grant read,write on directory impdp_dir to system;

3) 在控制台环境，在系统目录中创建目录 mkdir -p /backup/oracledata

4) 复制数据库备份文件至/backup/oracledata/目录

​ cp /u01/app/oracle/admin/orcl/dpdump/expdat.dmp /backup/oracledata

5）修改复制过来文件的权限

​ chown -R oracle:oinstall /backup/oracledata

6) 在控制台环境，恢复数据库(全库恢复) impdp system/Jiaminjm1@orcl DIRECTORY=impdp_dir DUMPFILE= expdat.dmp logfile= dmpfile.log full=y ignore=y

注意：

  1. logfile会在当前目录自动创建，不需要修改

问题：bash: Impdp: command not found…

解决：是因为没有切换用户，su - oracle，再次尝试

问题：oracle用户的权限不够

ORA-31640: unable to open dump file "/backup/oracledata/expdat.dmp" for read
ORA-27041: unable to open file
Linux-x86_64 Error: 13: Permission denied

解决：是因为复制过来的备份文件所有者是root，需要修改为oracle

​ chown -R oracle:oinstall /backup/oracledata

启动/关闭/重启

https://blog.csdn.net/heshushun/article/details/79773886

启动

  1. 先登录服务器后，切换到oracle用户状态下： su - oracle
  2. 检查oracle监听器运行状态： lsnrctl status命令查看。
    1. 若没启动，执行启动监听，通过命令： lsnrctl start （lsnrctl stop 关闭监听）
      1. 如果启动报错，listener.ora文件打开，调整HOST IP
  3. 进入sqlplus，命令： sqlplus /nolog
  4. 管理员 sysdba 权限登陆oracle： conn /as sysdba
  5. 启动服务（实例）： startup

关闭

关闭服务（实例）： shutdown immediate;

重启

SQL> shutdown immediate

SQL> startup open

检验服务是否开启

看数据库当前状态

SQL>select status from v$instance;

如果status = open 就说明oracle服务正常。
