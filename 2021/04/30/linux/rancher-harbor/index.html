Rancher+harbor

单主节点

虚拟机搭建

搭建两台主机，一台harbor、docker，一台rancher、docker

  1. cpu 2核，内存2G
  2. GUI –> development Tools
  3. 分区：/boot 200M,swap 4G,/
  4. 主机1：
    1. IP：192.168.42.241
      主机名harbor
  5. 主机2：
    1. IP：192.168.42.242
      主机名rancher02

安装

  1. 修改主机名 hostnamectl set-hostname rancher

  2. 修改IP和UUID

    uuidgen ens33
    记录序列号
vi /etc/sysconfig/cetwork-scripts/ifcfg-ens33
    修改IP和UUID
systemctl restart network

  3. 配置免密登录

    ssh-keygen -t rsa -b 2048回⻋ 回⻋ 回⻋

    ssh-copy-id root@192.168.42.241 # $IP 为所有节点地址包括⾃⾝，按照提⽰输⼊ yes 和 root 密码

    ssh-copy-id root@192.168.42.242 # $IP 为所有节点地址包括⾃⾝，按照提⽰输⼊ yes 和 root 密码

    在输入242之前，需要先创建出242的虚拟机，才可以进行设置

  4. 关闭防火墙

    systemctl stop firewalld #这个反应会比较慢

    systemctl disable firewalld

    systemctl status firewalld

  5. 关闭setlinux

    setenforce 0

    sed -i 's/SELINUX=enforcing/SELINUX=disabled/g' /etc/selinux/config

    getenforce

  6. 关闭swap

    cp -p /etc/fstab /etc/fstab.bak

    sed -i "s/\/dev\/mapper\/centos-swap/\#\/dev\/mapper\/centos-swap/g" /etc/fstab

    cat /etc/fstab

    swapoff -a

  7. 配置每台主机的hosts(/etc/hosts)

    方式一：
cat >> /etc/hosts<<EOF
172.16.83.190 harbor
192.168.42.241 rancher01
192.168.42.242 rancher01

方式二：
vim /etc/hosts
在末尾追加这几条信息

    提示：

    方式一有追加和替换两种形式，多用于shell脚本中，这里科普一下，后面用得到

    cat >> xxx <<EOF 为追加
cat >  xxx <<EOF 为替换

  8. 设置主机时区（如果在配置虚拟机时已经设置，这里可以跳过）

    date -R 或者 timedatectl

    ln -sf /usr/share/zoneinfo/Asia/Shanghai /etc/localtime

    echo 'LANG="en_US.UTF-8"' >> /etc/profile;source /etc/profile

  9. Kernel性能调优，数值根据环境自行配置

    方式一:
cat >> /etc/sysctl.conf<<EOF
net.ipv4.ip_forward=1
net.bridge.bridge-nf-call-iptables=1
net.ipv4.neigh.default.gc_thresh1=4096
net.ipv4.neigh.default.gc_thresh2=6144
net.ipv4.neigh.default.gc_thresh3=8192
EOF

方式2：
vim /etc/sysctl.conf
末尾追加信息

    sysctl -p

    如果报错：

    sysctl: cannot stat /proc/sys/net/bridge/bridge-nf-call-iptables: No such file or directory

    解决：

    modprobe br_netfilter

    reboot#重启系统

    再次尝试sysctl -p

安装docker

离线安装docker

由于自己是桥接网络，只能访问内网不能访问外网，因此不能使用在线docker

  1. 上传docker-18.09.9.tgz和wokinst_rancher_rke04_docker.sh到/opt目录

    这里有两个坑

    1. 文件和文档中的名字不一样
      1. E:\公司培训\rancherha\rancherha\soft\down\docker-18.09.9.tgz
      2. E:\公司培训\rancherha\rancherha\soft\wokinst_rancher_rke04_docker.sh
    2. 必须将这两个文件放到/opt目录下

  2. 执行脚本

    sh /opt/install_docker.sh

  3. 开启

    systemctl start docker && systemctl enable docker && docker version

  4. 查看docker是否完成安装

    docker ps，出现一行英文，类似列名，就证明成功了

    也可以docker -v

  5. 修改registry-mirrors，insecure-registries

    方式一：（此时是1个“>”，所以是替换操作）
cat > /etc/docker/daemon.json<<EOF
{
    "oom-score-adjust": -1000,
    "log-driver": "json-file",
    "log-opts": {
    "max-size": "100m",
    "max-file": "3"
    },
    "max-concurrent-downloads": 10,
    "max-concurrent-uploads": 10,
    "registry-mirrors": ["https://docker.mirrors.ustc.edu.cn"],
    "insecure-registries": ["http://192.168.42.241"],
    "storage-driver": "overlay2",
    "storage-opts": [
    "overlay2.override_kernel_check=true"
    ]
}
方式二：
vim /etc/docker/daemon.json
按照上面修改信息

  6. 重启docker

    systemctl daemon-reload && systemctl restart docker && docker version

安装docker-compose

  1. 上传文件v1.25.5-docker-compose-Linux-x86_64到/opt目录下
    1. E:\公司培训\rancherha\rancherha\soft\bin\v1.25.5-docker-compose-Linux-x86_64
  2. mv /opt/v1.25.5-docker-compose-Linux-x86_64 /usr/local/bin/docker-compose
  3. chmod +x /usr/local/bin/docker-compose
  4. docker-compose --version

安装私有镜像仓库harbor

  1. 上传harbor-offline-installer-v1.9.3.tgz文件到/opt目录

    1. E:\公司培训\rancherha\rancherha\soft\down\harbor-offline-installer-v1.9.3.tgz

  2. 解压

    tar -zxvf /opt/harbor-offline-installer-v

    vim /opt/harbor/harbor.yml

    hostname: 192.168.42.241
http:
port: 80

harbor_admin_password: newcapec

  3. 安装

    cd /opt/harbor

    ./install.sh*会自动导入镜像并启动对应的容器

    docker-compose ps，查看镜像的启动情况，是否都是up

    如果镜像没启动，可以

    docker-compose start

    如果修改了配置文件，可以重启

    ./prepare

    docker-compose up -d

    扩展：docker-compose 使用 ：

    docker-compose ps -a ###查看 Compose 应用中的各个容器。

    docker-compose up -d ###后台启动，如果容器不存在根据镜像自动创建并启动（用于修改了配置文件）

    docker-compose down -v ###停止容器并删除容器

    docker-compose start ###启动容器，容器不存在就无法启动，不会自动创建镜像

    docker-compose stop ###停止容器

    docker-compose restart ###重启容器

    其中操作的所有容器，默认情况下docker-compose就是操作同目录下的docker-compose.yml文件

  4. 登录

    如果网页无法访问，说明修改了配置文件或重启了机器，使用上边./prepare和docker-compose up -d重启，

    使用docker-compose ps查看启动情况

    即可访问

    http://192.168.42.241

    用户名：admin，密码就是配置文件中的harbor_admin_password，即newcapec

    1. 建立项目wymproject，设置公开

  5. 所有服务器需要配置镜像仓库并重启docker

    1. 配置镜像仓库（和上面的一个步骤一样，我配置的是第一个vancher主机，其他主机也要配置）

      方式一：（替换）
cat > /etc/docker/daemon.json<<EOF
{
"registry-mirrors": ["https://docker.mirrors.ustc.edu.cn"],
"insecure-registries": ["http://192.168.42.242"],
}
EOF

方式二：
vim /etc/docker/daemon.json
修改以上信息

    2. 重启docker

      systemctl daemon-reload && systemctl restart docker

  6. 所有服务器可以登录验证

    docker login 192.168.42.241（harbor的主机IP）

    Username:admin

    Password:newcapec

  7. docker拉取推送镜像

    1. 制作镜像标签：docker tag nginx:1.15.9-alpine 192.168.42.241/wymproject/nginx:1.14.2-alpine

  8. 推送镜像：docker push 192.168.42.241/wymproject/nginx:1.14.2-alpine

格式：

  1. 制作镜像标签：docker tag 镜像名[:标签名] harbor的IP/仓库名/镜像名[:标签名]
  2. 推送镜像：docker push harbor的IP/仓库名/镜像名[:标签名]

这里会遇到错误，意思是docker仓库中找不到nginx:1.15.9-alpine这个镜像。

解决：

方式一：查找docker仓库中有哪些nginx，拉取有的Nginx

docker search nginx

docker tag nginx 192.168.42.241/wymproject/nginx-alpine

方式二：找个本地的nginx，将其转换为镜像，并将其打标签推送

cat nginx-1.18.0-alpine.tar | docker import - nginx:1.18.0-alpine

docker images查看镜像nginx是否已经被拉取

docker tag nginx:1.18.0-alpine 192.168.42.241/wymproject/nginx:1.18.0-alpine

docker push 192.168.42.241/wymproject/nginx:1.18.0-alpine

  1. 查看效果

    登录192.168.42.241

Habor配置HTTPS

在/opt/harbor中执行以下命令，注意里面的IP需要修改

具体参考：

https://www.cnblogs.com/cjwnb/p/13441071.html

https://www.bilibili.com/video/BV17J411y7mW?from=search&seid=4694057997034590851

openssl genrsa -out ca.key 4096

openssl req -x509 -new -nodes -sha512 -days 3650 \
 -subj "/C=CN/ST=Beijing/L=Beijing/O=example/OU=Personal/CN=192.168.42.241" \
 -key ca.key \
 -out ca.crt

openssl genrsa -out 192.168.42.241.key 4096

openssl req -sha512 -new \
    -subj "/C=CN/ST=Beijing/L=Beijing/O=example/OU=Personal/CN=192.168.42.241" \
    -key 192.168.42.241.key \
    -out 192.168.42.241.csr

cat > v3.ext <<-EOF
authorityKeyIdentifier=keyid,issuer
basicConstraints=CA:FALSE
keyUsage = digitalSignature, nonRepudiation, keyEncipherment, dataEncipherment
extendedKeyUsage = serverAuth
subjectAltName = IP:192.168.42.241
EOF

openssl x509 -req -sha512 -days 3650 \
    -extfile v3.ext \
    -CA ca.crt -CAkey ca.key -CAcreateserial \
    -in 192.168.42.241.csr \
    -out 192.168.42.241.crt

mkdir -p /data/cert/

cp 192.168.42.241.crt /data/cert/

cp 192.168.42.241.key /data/cert/

openssl x509 -inform PEM -in 192.168.42.241.crt -out 192.168.42.241.cert

mkdir -p /etc/docker/certs.d/192.168.42.241/

cp 192.168.42.241.cert /etc/docker/certs.d/192.168.42.241/

cp 192.168.42.241.key /etc/docker/certs.d/192.168.42.241/

cp ca.crt /etc/docker/certs.d/192.168.42.241/

vim harbor.yml
https:
  port: 443
  certificate: /data/cert/192.168.42.241.crt 
  private_key: /data/cert/192.168.42.241.key

./prepare

docker-compose down -v

docker-compose up -d

访问：https://192.168.42.241

注意：

  1. docker-compose up -d启动时，如果出现以下情况，证明端口被占用了

    Error starting userland proxy: listen tcp 0.0.0.0:443端口: bind: address already in use端口占用问题
或
ERROR: for nginx Cannot start service proxy: driver failed programming-markdown

    这里我显示的是443端口被占用。（被占用的原因在于我把harbor和rancher放在了同一个机器中，rancher创建的集群会和harbor冲突，因为我们不要把之后的rancher部署在harbor机器中）

    docker ps -a查看对应端口。

    docker stop 服务id停掉或修改端口。修改运行中的容器方法在docker文章中。

  1. 配置后Https之后，访问harbor的网址必须要带https，https://192.168.42.241，

    因为我们访问这个网址默认端口是80，harbor的配置文件有写到，当我们配置了Https时，默认端口就会重定向到https的端口443

准备rancher离线安装镜像并导入harbor私有镜像仓库

  1. 登录harbor页面，创建rancher仓库

  2. 将rancher安装脚本和离线镜像上传/opt/rancher-v2.3.6中

  3. 下载离线安装包（镜像）

    如果有3.57G的镜像文件rancher-images.tar.gz，可以跳过1，2步，直接进行第3步

    1. 设置阿里云docker源

      目的是解决下载镜像文件过慢的问题

      1. vim /etc/docker/daemon.json

      2. 修改如下配置

        “registry-mirrors”:[“https://9cpn8tt6.mirror.aliyuncs.com”,”https://registry.docker-cn.com”

      3. 重启docker

        sudo systemctl daemon-reload
sudo systemctl restart docker

    2. sh rancher-save-images.sh --image-list ./rancher-images.txt

      此过程时间会很长，而且下载的大小越15G，注意磁盘空间

    3. 此时目录下会出现一个3.57G的tar包rancher-images.tar.gz。导入镜像仓库

      1. 登录harbor镜像仓库

        docker login ip或域名

      2. 导入到镜像仓库

        sh rancher-load-images.sh --image-list ./rancher-images.txt --images ./rancher-images.tar.gz --registry 172.16.42.241

    4. 查看效果

      登录harbor页面：192.168.42.241

部署Rancher

强烈建议：

  1. 用两台服务器，harbor和rancher各放一个，因为harbor开启了https之后会产生一个443端口，和之后的rancher集群有冲突，因此不能装在一起

开启另一台虚拟机，安装docker。

在线部署rancher需要有外网条件，就是Ping baidu.com。

执行docker run -d --restart=unless-stopped -p 81:80 -p 443:443 rancher/rancher:v2.3.6（这里可以用80:80）

意思是：将rancher镜像加载起来，以81端口映射rancher的80端口。我们访问192.168.42.241:81其实就是访问的rancher80端口。

错误：找不到镜像

解决：

  1. 从harbor仓库中拉取镜像

    1. 登录harbor：docker login 192.168.42.242，harbor账号密码
    2. 在harbor页面中找到对应的镜像，docker pull 192.168.42.241:80/rancher/rancher:v2.3.6
    3. 查看本地镜像：docker images

  2. 运行镜像

    1. docker run -d --restart=unless-stopped -p 80:80 -p 443:443 b821，b821是镜像id

  3. 访问：192.168.42.242:80

    扩展：

    1. 查看镜像docker images
    2. 删除镜像docker rmi images_id，强制删除-f
    3. 运行镜像docker run images_id
    4. 查看容器docker ps，查看所有（运行+原有）-a
    5. 运行容器docker start/stop 容器id
    6. 删除容器docker rm 容器id

错误：Docker容器做端口映射报错 docker: Error response from daemon: driver failed programming external connectivity

解决：systemctl restart docker（重新启动docker）

错误：如何删除和新建容器

解决：

  1. 停掉容器rancher，并删除该容器

    docker ps查看正在运行的容器

    docker stop 168停止容器名为name/id的容器（168是容器Id的前三位，系统可以自动找到该容器）

    docker rm 168删除容器（如果删除失败，就docker rm -f 168强制删除）

  2. 重启harbor

    docker-compose start
# 如果修改了配置⽂件，然后重启
./prepare
docker-compose up -d
docker-compose ps 查看状态

  3. 访问测试：192.168.42.241:81

    1. 如果不能访问成功，清理浏览器缓存重试

  4. 新建容器

    docker run -d --restart=unless-stopped -p 81:80 -p 443:443 rancher/rancher:v2.3.6

查看效果：

  1. 访问rancher：192.168.42.241:81

    如果报错：

    Client sent an HTTP request to an HTTPS server.

    解决：

    方式一：访问地址改为：https://192.168.42.241:443

建立集群
